# Tech Spec: Plataforma de Inversion en Renta Fija

> **Origen:** CB-001 (restricciones: equipo .NET/AWS, presupuesto $1.200M),
> RFC-001 (opcion C aprobada), ADR-001 (event-driven), PRD-001 (casos de uso)
>
> **Nota de madurez:** Este artefacto esta en Review. Los contratos con sistemas
> externos dependen de negociacion en curso (ver 🔴 TODOs).

## 3. Contratos de Integracion

> Cada contrato define QUE se intercambia y con que SLA.
> La implementacion (librerias, middleware, serializers) es responsabilidad del SWA.

### Contrato: Web App -> Investment API
| Campo | Valor |
|---|---|
| Protocolo | REST/HTTPS |
| Autenticacion | JWT (expiracion 15 min) — politica definida en CB-001 |
| Endpoints principales | POST /inversiones, GET /inversiones/{id}, GET /portafolio |
| Respuesta a creacion | 202 Accepted {id, estado: "Recibida"} (async por ADR-001) |
| NFR aplicable | QA-001 del PRD-001: latencia < 500ms p99 |

### Contrato: Investment API -> Cola SQS
| Campo | Valor |
|---|---|
| Protocolo | AWS SDK (async) |
| Formato | JSON |
| Idempotencia | Idempotency-Key en header (UUID v4, generado por cliente) |
| DLQ | Configurada — politica: max 3 reintentos con backoff exponencial |

### Contrato: Worker -> Deceval (Custodia)
| Campo | Valor |
|---|---|
| Proveedor | Deceval |
| Protocolo | REST/HTTPS |
| SLA contratado | 99% disponibilidad, latencia < 10s |
| Timeout | 15 segundos |
| Patron de integracion | Anti-Corruption Layer (ver CM-001) |
| Fallback | Enqueue para reintento en T+5min, usuario ve "En Proceso" |

## 4. Quality Attribute Trade-offs

> Trade-offs documentados como decisiones del SA. Cada uno referencia
> el NFR del PRD o la restriccion del CB que lo origina.

| Trade-off | Decision | Justificacion | Origen |
|---|---|---|---|
| Consistencia vs UX | Eventual consistency + feedback inmediato | El usuario ve "En Proceso" → "Confirmada". Mejor UX que esperar 30s | ADR-001, QA-002 (PRD) |
| Simplicidad vs Resiliencia | Event-driven con SQS | Resiliente a fallos de proveedores externos (Deceval, PSE) | ADR-001, CB-001 (sistemas externos) |
| Costo vs Escalabilidad | ECS Fargate (pay-per-use) | Dentro de presupuesto CB-001. Escala a 0 en baja demanda | CB-001 ($1.200M) |
