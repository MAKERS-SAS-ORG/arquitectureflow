## 3. Gates de Revision Arquitectonica

### Gate 1: Sprint Planning

**Cuando:** Al inicio de cada sprint (quincenal)
**Quien:** SA + Tech Lead
**Que revisar:**

- [ ] Las historias del sprint estan alineadas con el scope del PRD-001
- [ ] Si hay decisiones tecnicas nuevas, existe ADR o se necesita uno
- [ ] Los bounded contexts del CM-001 se respetan en la asignacion de trabajo
- [ ] No hay conflictos con restricciones de CB-001

### Gate 2: Code Review Arquitectonico

**Cuando:** PRs que tocan: limites de bounded context, integraciones externas, seguridad, o audit trail
**Quien:** SA o Tech Lead como delegado arquitectonico
**Que revisar:**

- [ ] El PR respeta los contratos de integracion definidos en TS-001 seccion 3
- [ ] No hay dependencias directas entre bounded contexts (viola CM-001)
- [ ] Patrones de seguridad siguen STRIDE del SD-001
- [ ] Si hay desviacion, esta documentada en seccion 4 de este tablero

### Gate 3: Pre-Deploy

**Cuando:** Antes de cada deploy a staging/produccion
**Quien:** Automatizado (CI/CD) + SA en primer deploy
**Que revisar:**

- [ ] FF-001 (latencia k6): PASS
- [ ] FF-002 (vulnerabilidades Snyk/SAST): PASS
- [ ] FF-003 (audit trail E2E): PASS
- [ ] FF-004 (aislamiento datos): PASS
- [ ] FF-006 (desacoplamiento NetArchTest): PASS
- [ ] Checklist de RO-001 seccion 2 cumplido

### Gate 4: Sprint Review / Mensual

**Cuando:** Sprint review o mensualmente
**Quien:** SA + Tech Lead + Ops
**Que revisar:**

- [ ] SLOs del SD-001 se estan cumpliendo (99.5% disponibilidad)
- [ ] FF-005 (disponibilidad monitoring): dentro de umbrales
- [ ] FF-007 (juicio IA): ejecutar analisis de coherencia de artefactos
- [ ] No hay deuda arquitectonica no documentada
- [ ] Actualizar este tablero con el estado real

### Flujo de desviacion

```
Desarrollador detecta que no puede seguir la arquitectura
    |
    v
Documenta en este registro (D-NNN)
    |
    v
Tech Lead evalua impacto
    |
    +-- Impacto Bajo -> Aceptar, documentar, continuar
    |
    +-- Impacto Medio -> Discutir con SA, posible actualizacion de artefacto
    |
    +-- Impacto Alto -> SA evalua, genera nuevo ADR, actualiza artefactos afectados
```
