# Context Map: Plataforma de Inversion en Renta Fija

> Referencia: Evans, E. *Domain-Driven Design.* 2003. Capitulo 14.

## Bounded Contexts Identificados

| Bounded Context | Subdominio | Tipo | Responsabilidad |
|---|---|---|---|
| **Identidad** | Supporting | Interno | Registro, KYC, autenticacion, perfil del inversionista |
| **Inversiones** | Core | Interno | Catalogo de CDTs, ordenes de compra, portafolio, rendimientos |
| **Pagos** | Supporting | Interno (integra externo) | Debito/credito via PSE, reconciliacion |
| **Custodia** | Generic | Externo (Deceval) | Registro y liquidacion de titulos valores |
| **Compliance** | Supporting | Interno | Audit trail, reportes SFC, monitoreo transaccional |

## Relaciones entre Contexts

| Upstream | Downstream | Patron | Descripcion |
|---|---|---|---|
| **Identidad** | **Inversiones** | Customer-Supplier | Identidad provee datos del usuario verificado; Inversiones consume |
| **Inversiones** | **Pagos** | Customer-Supplier | Inversiones solicita debito/credito; Pagos ejecuta |
| **Inversiones** | **Custodia (Deceval)** | Anti-Corruption Layer | Inversiones traduce su modelo al API de Deceval via ACL |
| **Pagos** | **PSE/ACH** | Conformist | Pagos se adapta al protocolo PSE tal como esta |
| **Inversiones** | **Compliance** | Published Language | Inversiones publica eventos de dominio; Compliance los consume para audit trail |
| **Identidad** | **Proveedor KYC** | Anti-Corruption Layer | Identidad traduce respuesta del proveedor KYC a su modelo |

## Justificacion de Patrones

### Anti-Corruption Layer (ACL) con Deceval
Deceval tiene un API legacy con modelo de datos propio (titulos, depositos, cuentas de custodia).
El ACL traduce entre el lenguaje del dominio de Inversiones (ordenes, portafolio, rendimientos)
y el lenguaje de Deceval (depositos, transferencias de titulos). Esto protege al dominio core
de cambios en el API externo.

### Conformist con PSE
PSE es un estandar nacional sin negociacion posible. El contexto de Pagos se adapta al protocolo
PSE tal como esta definido. No hay margen para negociar contratos.

### Published Language con Compliance
Inversiones publica eventos de dominio (InversionCreada, InversionConfirmada, InversionVencida)
en un formato estandar que Compliance consume para construir el audit trail.
Esto desacopla la logica de negocio de los requisitos regulatorios.

## Domain Events entre Contexts

| Evento | Productor | Consumidor | Payload principal |
|---|---|---|---|
| UsuarioVerificado | Identidad | Inversiones | userId, kycStatus, fechaVerificacion |
| InversionSolicitada | Inversiones | Pagos | inversionId, monto, cuentaOrigen |
| PagoConfirmado | Pagos | Inversiones | pagoId, inversionId, montoDebitado |
| PagoFallido | Pagos | Inversiones | pagoId, inversionId, motivo |
| InversionRegistrada | Inversiones | Compliance | inversionId, userId, monto, cdtId, timestamp |
| InversionConfirmada | Inversiones | Compliance | inversionId, custodiaRef, timestamp |
| InversionVencida | Inversiones | Pagos, Compliance | inversionId, montoCapital, rendimiento |
