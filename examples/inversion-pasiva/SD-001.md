# System Design: Plataforma de Inversion en Renta Fija

> Diagrama C4 L2 Container + Deployment logico.

## 2. Drivers de Arquitectura

### 2.1 Requisitos de Rendimiento

| Escenario | Latencia objetivo | Throughput | Medicion |
|---|---|---|---|
| Consultar portafolio | < 500ms P99 | 400 req/hora pico | p99 en CloudWatch |
| Crear inversion (API) | < 1s P99 (202 Accepted) | 80 req/hora pico | p99 en CloudWatch |
| Proceso E2E de inversion | < 5 min | 80 inversiones/hora | CloudWatch custom metric |
| Onboarding + KYC | < 60s verificacion | 200/hora pico | p99 en CloudWatch |

### 2.2 Disponibilidad y Tolerancia a Fallos

| Componente | Disponibilidad | Estrategia |
|---|---|---|
| Investment API (ECS Fargate) | 99.5% | Multi-AZ, health checks, auto-scaling |
| PostgreSQL (RDS) | 99.95% | Multi-AZ, automated backups |
| Redis (ElastiCache) | 99.5% | Cluster mode, Multi-AZ |
| SQS | 99.99% | Managed, sin accion requerida |

**RPO:** 1 hora (backups continuos de RDS)
**RTO:** 4 horas (failover Multi-AZ + redeploy si es necesario)

### 2.3 Escalabilidad

| Horizonte | Inversiones/hora | Usuarios | Estrategia |
|---|---|---|---|
| Lanzamiento | 80 | 10.000 | 2 ECS tasks, RDS db.r6g.large |
| 1 ano | 200 | 50.000 | Auto-scaling 2-6 tasks, read replica |
| 3 anos | 500 | 200.000 | Evaluar microservicios (ver ADR futuro) |

Trigger de escalado: CPU > 70% o SQS queue depth > 100 por mas de 3 minutos.

## 3. Seguridad (STRIDE)

| Amenaza | Riesgo | Mitigacion |
|---|---|---|
| **Spoofing** (suplantacion) | Alto | JWT con expiracion 15min + MFA en operaciones financieras |
| **Tampering** (manipulacion) | Critico | Audit trail inmutable con hash encadenado, HTTPS/TLS 1.3 |
| **Repudiation** (repudio) | Critico | Audit trail con timestamp + hash, no es modificable |
| **Information Disclosure** | Alto | Cifrado en reposo (AES-256) + en transito (TLS), RBAC por usuario |
| **Denial of Service** | Medio | AWS WAF + rate limiting en ALB (100 req/min por IP) |
| **Elevation of Privilege** | Alto | Principio minimo privilegio, cada usuario solo ve sus datos (RN-002) |

### Flujo de seguridad
```
Internet -> CloudFront -> AWS WAF -> ALB -> ECS (VPC privada)
                                             |
                                   RDS / ElastiCache / SQS
                                   (subredes privadas, sin acceso publico)
```

## 4. Dependencias Externas

| Sistema | SLA contratado | Timeout | Fallback |
|---|---|---|---|
| Deceval (custodia) | 99% | 15 seg | Enqueue reintento T+5min, notificar usuario |
| PSE/ACH (pagos) | 99.5% | 30 seg | Reintento auto x2, luego notificar usuario |
| Proveedor KYC | 99% | 10 seg | Fallback a verificacion manual (cola de revision) |
| SFC (reportes) | Sin SLA real-time | 60 seg | Reportes batch diarios, no critico en real-time |

## 5. Observabilidad (SA define QUE medir)

**Plataforma elegida:** AWS CloudWatch + X-Ray (ver ADR futuro)

### Metricas criticas
| Metrica | Umbral de alarma | Por que importa |
|---|---|---|
| Disponibilidad API | < 99.5% en 1 hora | SLO comprometido |
| Latencia P99 API | > 1s | Degradacion UX |
| Tasa de error API | > 1% en 5 min | Posible incidente |
| SQS queue depth | > 200 por 5 min | Backlog excesivo, procesamiento lento |
| Deceval error rate | > 20% en 10 min | Proveedor posiblemente caido |
| Tasa de conversion KYC | < 70% en 1 dia | Posible problema en flujo de onboarding |

## 6. SPOFs (Single Points of Failure)

| SPOF | Impacto | Mitigacion actual | Pendiente |
|---|---|---|---|
| Deceval | No se pueden registrar inversiones | Reintento auto + notificacion | Negociar SLA formal |
| PSE/ACH | No se pueden debitar/acreditar fondos | Reintento auto x2 | Evaluar pasarela alternativa |
| Proveedor KYC | No se pueden registrar nuevos usuarios | Cola de verificacion manual | Segundo proveedor |

## 7. Capacity Planning (SA define umbrales)

| Recurso | Actual | Proyeccion 6m | Trigger de escalado | Estrategia |
|---|---|---|---|---|
| ECS Tasks | 2 | 4 | CPU > 70% por 3 min | Auto-scaling horizontal |
| RDS storage | 50 GB | 150 GB | > 80% usado | Storage auto-scaling ON |
| RDS instance | db.r6g.large | db.r6g.xlarge | CPU > 80% por 5 min | Vertical scaling |
| SQS | ilimitado | ilimitado | N/A | Managed |
