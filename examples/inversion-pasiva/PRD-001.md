# PRD: Plataforma de Inversion en Renta Fija

> **Origen:** CB-001 (scope IN/OUT), RFC-001 (opcion C aprobada)
> **Nota de madurez:** Este PRD esta en Review. Las secciones marcadas con 🔴 TODO
> requieren validacion con el Product Owner antes de pasar a Approved.

## 1. Objetivo del Producto

Permitir que personas naturales colombianas (usuario identificado en CB-001) inviertan
en instrumentos de renta fija (CDTs y bonos) desde $100.000 COP, con un proceso de
onboarding digital menor a 10 minutos.

## 2. Usuarios Objetivo

| Persona | Descripcion | Necesidad Principal |
|---|---|---|
| Inversionista novato | Persona con capacidad de ahorro < $5M COP, sin experiencia en inversiones | Invertir de forma simple y segura sin intermediarios |
| Inversionista recurrente | Persona que ya uso la plataforma y quiere ampliar su portafolio | Ver rendimiento, reinvertir, diversificar |
| Oficial de compliance | Equipo interno que monitorea operaciones | Audit trail completo, reportes regulatorios |

## 3. Funcionalidades (Casos de Uso)

### CU-001: Onboarding digital
- **Actores:** Inversionista novato
- **Precondicion:** El usuario tiene cedula colombiana vigente y cuenta bancaria
- **Flujo principal:**
  1. El usuario se registra con email y telefono
  2. El sistema solicita datos personales y foto de cedula
  3. El sistema ejecuta verificacion KYC (identidad + listas restrictivas)
  4. El sistema crea la cuenta del inversionista
  5. El usuario vincula su cuenta bancaria (PSE)
- **Flujos alternativos:**
  - Alt-A: KYC rechazado -> notificar al usuario con motivo generico, no revelar detalles de listas
  - Alt-B: Cuenta bancaria no verificable -> permitir reintento con otro banco
- **Postcondicion:** Usuario con cuenta activa, KYC aprobado, cuenta bancaria vinculada
- **Prioridad:** Must Have

### CU-002: Comprar inversion en CDT
- **Actores:** Inversionista (novato o recurrente)
- **Precondicion:** Cuenta activa, KYC aprobado, cuenta bancaria vinculada
- **Flujo principal:**
  1. El usuario navega el catalogo de CDTs disponibles
  2. El usuario selecciona un CDT (emisor, plazo, tasa)
  3. El usuario ingresa monto (minimo $100.000 COP)
  4. El sistema muestra resumen: monto, tasa, plazo, rendimiento estimado
  5. El usuario confirma la inversion
  6. El sistema responde 202: "Inversion recibida, procesando"
  7. El sistema debita fondos via PSE
  8. El sistema registra la inversion en custodia (Deceval)
  9. El sistema notifica al usuario: "Inversion confirmada"
- **Flujos alternativos:**
  - Alt-A: PSE falla -> reintentar automaticamente x2, luego notificar "No pudimos debitar"
  - Alt-B: Deceval no disponible -> encolar para reintento, notificar "En proceso, te confirmaremos"
- **Postcondicion:** Inversion registrada en custodia, fondos debitados, usuario notificado
- **Prioridad:** Must Have

### CU-003: Consultar portafolio
- **Actores:** Inversionista recurrente
- **Precondicion:** Al menos una inversion activa
- **Flujo principal:**
  1. El usuario accede a su portafolio
  2. El sistema muestra: inversiones activas, rendimiento acumulado, proximos vencimientos
- **Postcondicion:** El usuario ve el estado actual de sus inversiones
- **Prioridad:** Must Have

### CU-004: Redimir inversion al vencimiento
- **Actores:** Inversionista, Sistema (automatico)
- **Precondicion:** CDT llega a fecha de vencimiento
- **Flujo principal:**
  1. El sistema detecta vencimiento del CDT
  2. El sistema solicita liquidacion a Deceval
  3. El sistema transfiere capital + rendimiento a cuenta bancaria del usuario via ACH
  4. El sistema notifica al usuario: "Tu inversion vencion, recibes $X"
- **Flujos alternativos:**
  - Alt-A: ACH falla -> reintentar en T+1, notificar al usuario
- **Postcondicion:** Fondos en cuenta del usuario, inversion cerrada
- **Prioridad:** Must Have

## 4. User Stories con Criterios de Aceptacion

### US-001: Registro e identidad
**Como** inversionista novato, **quiero** registrarme con mi cedula y verificar mi identidad,
**para** poder invertir de forma segura y regulada.

```gherkin
Scenario: Registro exitoso con KYC aprobado
  Given un usuario con cedula colombiana vigente
  When  completa el formulario de registro y sube foto de cedula
  Then  el sistema verifica identidad en menos de 60 segundos
  And   crea la cuenta con estado "Activa"
  And   envia confirmacion por email y SMS

Scenario: KYC rechazado por listas restrictivas
  Given un usuario que aparece en listas restrictivas
  When  completa el formulario de registro
  Then  el sistema rechaza el registro
  And   muestra mensaje generico "No podemos crear tu cuenta en este momento"
  And   NO revela el motivo especifico del rechazo
```

### US-002: Comprar CDT
**Como** inversionista, **quiero** comprar un CDT seleccionando emisor, plazo y monto,
**para** invertir mi dinero en renta fija con rendimiento predecible.

```gherkin
Scenario: Compra exitosa de CDT
  Given un usuario con cuenta activa y cuenta bancaria vinculada
  When  selecciona un CDT de $500.000 a 90 dias al 12% EA
  And   confirma la inversion
  Then  el sistema responde "Inversion recibida"
  And   debita $500.000 de su cuenta bancaria via PSE
  And   registra la inversion en Deceval
  And   notifica "Inversion confirmada" en menos de 5 minutos

Scenario: Monto menor al minimo
  Given un usuario que intenta invertir $50.000
  When  confirma la inversion
  Then  el sistema muestra "El monto minimo de inversion es $100.000 COP"
```

## 5. Quality Attribute Scenarios (NFRs)

### QA-001: Latencia de consulta de portafolio
| Campo | Valor |
|---|---|
| Fuente | Inversionista autenticado |
| Estimulo | Consulta su portafolio |
| Artefacto | API de Inversiones |
| Entorno | Operacion normal, hora pico |
| Respuesta | Retorna portafolio completo |
| Medida | Latencia < 500ms p99 |

### QA-002: Disponibilidad del flujo de compra
| Campo | Valor |
|---|---|
| Fuente | Inversionista autenticado |
| Estimulo | Inicia compra de CDT |
| Artefacto | API de Inversiones + Worker |
| Entorno | Operacion normal |
| Respuesta | Acepta la orden y la procesa E2E |
| Medida | Disponibilidad 99.5%, tiempo E2E < 5 minutos |

### QA-003: Seguridad de datos financieros
| Campo | Valor |
|---|---|
| Fuente | Atacante externo |
| Estimulo | Intenta acceder a datos de otro usuario |
| Artefacto | API de Inversiones |
| Entorno | Cualquiera |
| Respuesta | Rechaza con 403, registra intento en audit trail |
| Medida | Zero accesos no autorizados a datos de otros usuarios |

## 6. Reglas de Negocio

- RN-001: Monto minimo de inversion: $100.000 COP
- RN-002: Un usuario solo puede ver y operar sus propias inversiones (alineado con STRIDE en SD-001)
- RN-003: Toda operacion financiera debe quedar en audit trail inmutable (validado por FF-001 FF-003)
- RN-004: KYC debe completarse antes de cualquier operacion de inversion (compliance CB-001)
- RN-005: Los rendimientos se calculan segun tasa pactada al momento de la compra (no variable)
- RN-006: El usuario no puede retirar antes del vencimiento del CDT (excepto si el emisor lo permite)
- 🔴 TODO: RN-007 pendiente — Validar con Compliance el monto maximo por operacion sin aprobacion adicional
- 🔴 TODO: RN-008 pendiente — Validar con Product Owner si se permiten inversiones recurrentes automaticas

## 7. Fuera de Scope (explicito)

- **Inversion en renta variable (acciones)** — fase futura, diferente regulacion
- **Mercado secundario de CDTs** — complejidad regulatoria alta, evaluar en v2
- **App mobile nativa** — MVP es web responsive, app nativa en fase 2
- **Asesoria financiera automatizada (robo-advisor)** — requiere System Prompt Spec, fase futura
- **Multi-moneda** — solo COP en esta fase

## 8. Dependencias y Restricciones

- Depende de: Deceval (custodia), PSE/ACH (pagos), proveedor KYC
- Restricciones:
  - Sandbox regulatorio SFC debe estar aprobado (SUPUESTO-01 del RFC)
  - Ley 964/2005 (mercado de valores)
  - Ley 1581/2012 (datos personales)
  - Circular Basica SFC
