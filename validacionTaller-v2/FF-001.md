# Architecture Fitness Functions: AsistIA

## Roles colaboradores

> El SA define **que medir y umbrales**. :busts_in_silhouette: **Especialista Tecnico** implementa
> estaticas + dinamicas. :busts_in_silhouette: **DevOps** provee metricas continuas. :busts_in_silhouette: **QA**
> mapea cada FF a un QA-Scenario del PRD.

| Rol | Que aporta | Donde |
|---|---|---|
| **Especialista Tecnico** | Implementa NetArchTest, k6, contract tests | Modo Estatica + Dinamica |
| **DevOps** | CloudWatch dashboard, alertas Datadog | Modo Dinamica + dashboard |
| **QA** | Mapeo a QA-* del PRD, casos edge | Cobertura |

> Referencia: Ford, Parsons, Kua — *Building Evolutionary Architectures* (2nd Ed, 2023).

## Clasificacion por Modo

| Modo | Que valida | Cuando | Herramienta |
|---|---|---|---|
| **Estatica** | Reglas estructurales en codigo | Build / CI | NetArchTest, ESLint custom rules |
| **Dinamica** | Comportamiento bajo carga / produccion | CI load / continuous monitoring | k6, CloudWatch, Datadog |
| **Juicio** | Coherencia y vigencia de los .md arquitectonicos | Bajo demanda / Sprint review | Agente IA |

## Catalogo

### FF-001: Latencia chat conversacional
| Campo | Valor |
|---|---|
| Atributo | Performance (mapea PRD QA-001) |
| Tipo | Atomic |
| Modo | **Dinamica** |
| Trigger | k6 en CI (smoke + carga) + CloudWatch p95 continuo |
| Metrica | TTFT |
| Umbral | < 1.5s p95 / < 2.0s p99 |
| Herramienta | k6 + CloudWatch alarm |
| Accion si falla | Bloquear deploy si CI; alarma P2 si produccion |

### FF-002: PII redacted antes de LLM
| Campo | Valor |
|---|---|
| Atributo | Privacy / Compliance (mapea PRD RN-003) |
| Tipo | Atomic |
| Modo | **Estatica + Dinamica** |
| Trigger | Estatica: tests unitarios sobre `pal-llm`. Dinamica: sampler 1% de sesiones en produccion. |
| Metrica | % payloads con PII detectada al salir hacia LLM |
| Umbral | **0% — cualquier valor > 0 es violacion regulatoria** |
| Herramienta | Microsoft Presidio + custom regex tests + sampler en `pal-llm` |
| Accion si falla | **Bloquear deploy + page P1 (Compliance + Security)** |

### FF-003: Audit trail completo y WORM
| Campo | Valor |
|---|---|
| Atributo | Compliance (mapea PRD QA-004) |
| Tipo | Holistic |
| Modo | **Dinamica** |

| Metrica | (a) % sesiones con audit completo; (b) S3 Object Lock activo |
