## 2. Drivers de Arquitectura

### 2.1 Requisitos de Rendimiento

| Escenario | Latencia objetivo | Throughput | Medicion |
|---|---|---|---|
| Time-to-first-token (TTFT) chat | < 1.5s p95 | 1.000 sesiones concurrentes | k6 + CloudWatch |
| Latencia total respuesta | < 3s p95 | -- | k6 + CloudWatch |
| Consulta cobertura al Core | < 1.5s p95 | 200 req/seg | CloudWatch (X-Ray) |
| Generacion RIPS | < 30s | Batch 5k/dia | Worker metrics |

### 2.2 Disponibilidad y Tolerancia a Fallos

| Componente | Disponibilidad | Estrategia |
|---|---|---|
| `chat-api` | 99.5% | Multi-AZ, auto-scaling 2-20 tasks |
| `pal-llm` | 99.5% | Multi-AZ + failover Claude→GPT-4 |
| `claims-engine` | 99.5% | Multi-AZ |
| Core SOAP (externo) | 99% (legacy) | Cache de respuestas + fallback a "en revision" |
| Audit store | 99.99% | DynamoDB multi-AZ + S3 cross-region |

**RPO:** 5 minutos (audit trail no puede perder datos).
**RTO:** 15 minutos (failover regional).

### 2.3 Escalabilidad

| Horizonte | Metrica | Valor | Estrategia |

## 3. Seguridad (STRIDE)

> Validado con :busts_in_silhouette: **Security Officer** — sistema de salud, datos sensibles.

| Amenaza | Riesgo | Mitigacion |
|---|---|---|
| Spoofing | Alto | JWT RS256 + MFA opcional, mTLS interno, OAuth2 a externos |
| Tampering | Critico | TLS 1.3 obligatorio, hash SHA-256 de soportes, audit WORM |
| Repudiation | Critico | Audit trail inmutable WORM con timestamps + IP + agent (10 anos) |
| Information Disclosure | Critico | PII redaction antes de LLM, KMS para datos at-rest, principio de minimo privilegio |
| Denial of Service | Medio | WAF + rate limiting (60 msg/min/sesion) + AWS Shield |
| Elevation of Privilege | Alto | RBAC en evaluator-portal, segregacion de roles, audit de cambios de rol |

## 5. Observabilidad

> :busts_in_silhouette: **DevOps Lead** valido la lista. Plataforma: CloudWatch + X-Ray + Datadog (opcional).

### Metricas criticas que SA define

| Metrica | Umbral de alarma | Por que importa |
|---|---|---|
| Disponibilidad de `chat-api` | < 99.5% en 30 dias | SLO comprometido |
| TTFT chat | > 2s p95 sostenido 5 min | Degradacion UX |
| Tasa de fallback al proveedor secundario | > 5% sostenido 1h | Algo mal con primario |
| % sesiones con consentimiento NO firmado pero datos enviados | > 0 (cualquier valor) | **Violacion regulatoria — page** |
| % PII detectada DESPUES de redaction | > 0 (cualquier valor) | **Violacion RN-003 — page** |
| Auto-radicaciones revertidas por evaluador | > 8% en sprint | ADR-002 reglas mal calibradas |

### Lo que DevOps implementa (fuera de scope SA)
- Formato exacto de logs JSON.
- Configuracion de dashboards en CloudWatch.
- Retencion y rotacion (logs 90 dias, audit 10 anos).
- Alertas a PagerDuty.
