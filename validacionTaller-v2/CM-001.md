# Context Map: AsistIA

> Referencia: Evans 2003, Vernon 2016. C4 + DDD complementarios.

## Roles colaboradores

> :busts_in_silhouette: **Especialista Tecnico** — patrones DDD realistas y eventos de dominio.
> :busts_in_silhouette: **Acelerador (Dir. Operaciones)** — clasificacion Core/Supporting/Generic
> segun valor de negocio.

## Bounded Contexts Identificados

| Bounded Context | Subdominio | Tipo | Responsabilidad |
|---|---|---|---|
| **Conversacion** | Conversacion guiada con LLM | Core | Estado de la sesion, prompt versioning, PII redaction |
| **Reclamos** | Radicacion y orquestacion | Core | Crear, validar, enrutar reclamo |
| **Cobertura** | Lectura del Core de Polizas | Supporting | ACL — traduce SOAP del Core a domain model |
| **Evaluacion** | Decision humana de casos ambiguos | Core | Dashboard, asignacion, decision firmada |
| **Compliance / Audit** | Trazabilidad WORM 10 anos | Supporting | Append-only audit log |
| **Pagos** | Integracion con sistema de pagos interno | Generic | Iniciar liquidacion (no decide) |
| **Generacion RIPS** | Reporte regulatorio | Supporting | XML valido, batch a SuperSalud |

> Decision con :busts_in_silhouette: Acelerador: **Conversacion**, **Reclamos** y **Evaluacion** son
> Core (diferenciador competitivo). El resto son Supporting/Generic.

## Relaciones entre Contexts

| Upstream | Downstream | Patron | Descripcion |
|---|---|---|---|
| Core de Polizas (legacy) | Cobertura | **Anti-Corruption Layer (ACL)** | Cobertura traduce SOAP / modelo legacy a domain model limpio |
| Conversacion | Reclamos | **Customer-Supplier** | Conversacion crea draft de reclamo; Reclamos valida y persiste |
| Reclamos | Cobertura | **Customer-Supplier** | Reclamos consulta cobertura por session |
| Reclamos | Evaluacion | **Partnership** | Para casos ambiguos, ambos contextos coordinan estado |
| Reclamos | Compliance/Audit | **Open Host Service** | Reclamos publica eventos `ClaimStateChanged` que Audit consume |
| Reclamos | Pagos | **Customer-Supplier** | Reclamos solicita liquidacion |
| Reclamos | Generacion RIPS | **Published Language** | Evento `ClaimRadicado` con schema XSD versionado |

## Domain Events entre Contexts

| Evento | Productor | Consumidor | Payload principal |
|---|---|---|---|
| `ConsentimientoFirmado` | Conversacion | Compliance | sessionId, fecha, version texto, IP, agent |
| `ReclamoBorradorCreado` | Conversacion | Reclamos | sessionId, datos parciales |
| `CoberturaConsultada` | Cobertura | Reclamos | tipo (cubierto/ambiguo/negado), porcentaje |
| `ReclamoRadicado` | Reclamos | Generacion RIPS, Compliance, Pagos | reclamoId, asegurado, monto, prestador |
| `ReclamoEnrutadoEvaluador` | Reclamos | Evaluacion | reclamoId, contexto conversacional |
| `EvaluacionFirmada` | Evaluacion | Reclamos, Compliance | reclamoId, decision, evaluadorId, justificacion |
| `LiquidacionIniciada` | Reclamos | Pagos | reclamoId, monto, beneficiario |
