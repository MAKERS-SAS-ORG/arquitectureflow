## 3. Contratos de Integracion

### Contrato: `chat-api` -> `pal-llm`

| Campo | Valor |
|---|---|
| Protocolo | REST/HTTPS, JSON |
| Endpoint | `POST /v1/chat/completions` |
| Formato | OpenAI-compatible (compat con ambos proveedores) |
| Autenticacion | mTLS interno + JWT del usuario en header |
| SLA esperado | TTFT < 1.5s p95 (PRD QA-001) |
| Idempotencia | Por `Idempotency-Key` (header) |
| Estrategia de error | Failover automatico a proveedor secundario tras 1 timeout (5s); circuit breaker tras 3 fallos |

### Contrato: `claims-engine` -> Core de Polizas (sistema externo)

| Campo | Valor |
|---|---|
| Proveedor | Core de Polizas (legacy on-prem .NET FW 4.8) |
| Protocolo | SOAP/HTTPS via VPN |
| SLA contratado | 1.5s p95 (verificar con :busts_in_silhouette: **DevOps**) |
| Timeout configurado | 3s (con un retry idempotente) |
| Fallback | "Reclamo registrado en revision" (no bloquea al usuario) |

### Contrato: `claims-engine` -> Sistema de Pagos

| Campo | Valor |
|---|---|
| Proveedor | API interna (REST moderna) |
| Protocolo | REST/JSON |
| Autenticacion | OAuth2 client credentials |
| SLA | 99.9% (interno) |
| Timeout | 2s |
| Fallback | Queue para reintento async |

### Contrato: `rips-generator` -> SuperSalud RIPS endpoint (externo)

| Campo | Valor |
|---|---|
| Proveedor | MinSalud / SuperSalud |
| Protocolo | SFTP + XML |
| SLA | Batch diario (no real-time) |
| Idempotencia | Por `IdReclamo` |
| Estrategia de error | Retry hasta 3 veces; alerta a Compliance si falla |

## 4. Quality Attribute Trade-offs

| Trade-off | Decision | Justificacion |
|---|---|---|
| Consistencia vs Latencia (Core) | Eventual consistency aceptable | Si Core no responde en 3s, registramos reclamo offline y notificamos. UX > consistency en sesion conversacional. |
| Seguridad vs Complejidad | Maxima seguridad | Datos de salud + Habeas Data — no hay margen para shortcuts. |
| Vendor lock-in vs Time-to-market | Multi-vendor LLM | ADR-001 explicito. |
| Observabilidad detallada vs Costo | Detallada | Sistema de IA alto riesgo requiere trazabilidad — costo justificado. |

## 6. Politicas a Nivel de Solucion (SA define politica, SWA implementa)

| Politica | Decision SA | SWA implementa |
|---|---|---|
| Errores API | RFC 7807 Problem Details | Middleware ASP.NET, formato exacto |
| Versionamiento API | URL-based (`/v1/`, `/v2/`) | Routing, backward compat |
| Autenticacion usuario | JWT RS256, expiracion 15 min, refresh 8h | Middleware, refresh logic |
| Autenticacion servicio | mTLS interno + OAuth2 hacia externos | Cert rotation, vault |
| Secretos | AWS Secrets Manager, rotacion cada 90 dias | Integracion en deploy |
| Logging | JSON estructurado, correlation-id obligatorio, **PII redacted** | Formato exacto, structured fields |
| Idempotencia | Header `Idempotency-Key` en todas las escrituras | Storage de keys (Redis 24h), dedup |
| PII redaction antes de LLM | Regex + library (Microsoft Presidio) | Pipeline pre-LLM |
| Rate limiting | 60 mensajes/minuto por sesion | Token bucket, plataforma compartida |
