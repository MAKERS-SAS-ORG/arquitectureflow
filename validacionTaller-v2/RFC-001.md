# RFC-001: Canal Conversacional con LLM para Radicacion de Reclamos

## Roles colaboradores

> :busts_in_silhouette: **Acelerador** (ROI por opcion) + :busts_in_silhouette: **Especialista Tecnico**
> (viabilidad, riesgos). Ambos firmaron este RFC.

| Rol | Aporte concreto |
|---|---|
| **Acelerador** (Dir. Operaciones) | ROI por opcion (ahorro/reclamo, payback), criterios de adopcion |
| **Especialista Tecnico** (Tech Lead) | Viabilidad de integracion con Core legacy, riesgos LLM (alucinaciones, costos token), plan de rollback realista |
| **Compliance / Legal** | Confirmo que las 3 opciones son viables si hay human-in-the-loop |

## 1. Resumen Ejecutivo
> Construir un canal conversacional con LLM para que asegurados radiquen reclamos
> en menos de 10 minutos, integrado con Core de Polizas y RIPS, cumpliendo Habeas
> Data + SuperSalud 2266/2024.

## 2. Contexto y Motivacion

### 2.1 El Problema Hoy
- Tiempo medio de radicacion: 18 min (web/app).
- Tasa de abandono: 34%.
- Call center al 92% capacidad — overflow.

### 2.2 Costo de No Resolver
- Sobre-costo $340M COP/mes por uso de call center.
- Riesgo de multa SuperSalud > 1.000 SMMLV desde 2026-12-01.
- Perdida de NPS: -22 pts en segmento que radica reclamos.

### 2.3 Por Que Ahora
- **Resolucion 2266/2024** entra en vigor 2026-12-01 (deadline regulatorio).
- Modelos LLM 2025-2026 ya alcanzan calidad suficiente para conversaciones guiadas.
- Ventana competitiva de 6 meses antes que competidores (SURA, Colsanitas).

### 2.4 Metricas de Exito

| Metrica | Valor actual | Objetivo | Plazo |
|---|---|---|---|
| Tiempo medio radicacion | 18 min | < 5 min p95 | 2026-12-01 |
| Tasa de abandono | 34% | < 12% | 2027-03-31 |
| Reclamos via canal digital | 41% | > 75% | 2027-06-30 |
| NPS post-reclamo | 41 | > 60 | 2027-Q2 |

## 3. Opciones Consideradas

### Opcion A: No Hacer Nada
- **Descripcion:** Mantener call center + formularios web.
- **Pros:** $0 inversion. Sin riesgo de cambio.
- **Contras:** Multa SuperSalud certera. NPS sigue cayendo. CAC sube.
- **Estimacion:** $0 build / **$340M COP/mes overhead operativo + multas**.
- **Por que se descarta:** incumplimiento regulatorio inevitable.

### Opcion B: White-label de Vendor Conversacional (Glia, Conversocial, Drift)
- **Descripcion:** Comprar plataforma SaaS y configurarla. Sin LLM propio.
- **Pros:** Time-to-market 3-4 meses. Sin equipo LLM ops.
- **Contras:**
  - Vendor lock-in (datos de salud salen del pais — choque con Habeas Data).
  - Customizacion limitada para flujos RIPS y Core SOAP.
  - Costo recurrente alto (~$95M COP/mes a 50k sesiones).
  - :busts_in_silhouette: **Especialista Tecnico:** "ningun vendor evaluado integra con SOAP del Core sin middleware adicional".
- **Estimacion:** $400M COP build / $1.140M COP/ano operativo.
- **Por que se descarta:** vendor lock-in y datos fuera del pais → Compliance la rechaza.

### Opcion C: Plataforma Propia con LLM via API — RECOMENDADA
- **Descripcion:** Backend propio (.NET + AWS) que orquesta un LLM (Claude/GPT-4) via API,
  con guardrails (System Prompt Spec) y human-in-the-loop para decisiones sensibles.
- **Pros:**
  - Datos no salen de AWS Colombia (zona Sao Paulo + replica Bogota DC).
  - Control total sobre el system prompt y los flujos.
  - Talento del equipo (.NET + TypeScript) reutilizable.
  - Compatible con habilitar **2 proveedores LLM** para mitigar lock-in.
- **Contras:**
  - Time-to-market 7-9 meses.
  - Requiere capacitacion en LLM ops (eval, monitoring, prompt versioning).
  - Costo variable de tokens — sensible a uso.
- **Estimacion:** $1.350M COP build / $35M COP/mes operativo (incluye tokens).
- **Por que se recomienda:** unica viable para compliance + control + talento.

| Criterio | Peso | A: No hacer | B: Vendor | C: Propia |

## 5. Impacto y Riesgos

### 5.1 Sistemas y Modulos Afectados
- **Core de Polizas** (legacy SOAP): nueva integracion (consulta cobertura).
- **RIPS:** generacion automatizada del registro.
- **CRM Salesforce:** sincronizacion del historial.
- **Buzon S3:** carga de soportes medicos.

### 5.2 Riesgos con Plan de Mitigacion

| Riesgo | Probabilidad | Impacto | Mitigacion |
|---|---|---|---|
| **Alucinaciones del LLM** generan respuestas incorrectas sobre cobertura | Alta | Alto | Human-in-the-loop obligatorio antes de notificar al usuario; el LLM nunca decide sobre coberturas (consulta al Core) |
| **Costo de tokens** explota si el flujo no esta acotado | Media | Medio | Prompt caching + limit de turnos por sesion + presupuesto alerta automatica |
| **Latencia LLM** > 3s degrada experiencia conversacional | Media | Medio | Streaming de respuesta + fallback a siguiente proveedor LLM |
| **Filtracion de datos sensibles** al proveedor LLM | Baja | Critico | Anonymizacion en cliente + DPA firmado + opt-out de training |
| **Habeas Data:** consentimiento mal capturado | Media | Critico | Flujo de consentimiento explicito ANTES de iniciar conversacion (validado con Compliance) |

### 5.3 Plan de Rollback
> Activar feature flag `ai_assistant.enabled = false` redirige a flujo legacy (web form).
> Tiempo de rollback: < 2 minutos via dashboard. Sin perdida de datos (todas las
> conversaciones se persisten antes de cualquier accion irreversible).

### 5.4 Impacto Regulatorio
- **Aprobaciones requeridas:**
  - Compliance Officer (Habeas Data + tratamiento de datos sensibles).
  - Asesor regulatorio externo (SuperSalud) — opinion legal sobre IA en reclamos.
- **Reportes nuevos:** registro de tratamiento ante SIC (Superintendencia de Industria y Comercio).
