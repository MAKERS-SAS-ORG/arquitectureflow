Lee el archivo `skills/orquestador/SKILL.md` completo y ejecuta el flujo del orquestador desde la Fase 0.

El orquestador guia al arquitecto por 7 fases:
- Fase 0: Context Brief (OBLIGATORIO — sin este no hay nada mas)
- Fase 1: Seleccion de artefactos (segun references/matriz-decision.md)
- Fase 2-3: Creacion iterativa de artefactos (RFC → ADR → PRD → Tech Spec → System Design → Req. Operacionales)
- Fase 4: Critica automatica de cada artefacto (🔴🟡🟢)
- Fase 5: Diagramas C4 con Excalidraw MCP (skills/diagramas/SKILL.md)
- Fase 6: Verificacion de consistencia entre artefactos
- Fase 7: Tablero de Adherencia Arquitectonica (templates/tablero-adherencia.md)

Empieza SIEMPRE por la Fase 0: preguntale al usuario el contexto de negocio para generar el Context Brief usando `templates/context-brief.md`.

Guarda todos los artefactos en una subcarpeta del proyecto (ejemplo: `mi-proyecto/CB-001.md`).
