# RFC-001: Plataforma de Micro-Inversion en Renta Fija para Personas Naturales

## 1. Resumen Ejecutivo

Construir una plataforma digital propia que permita a personas naturales colombianas
invertir en CDTs desde $100.000 COP, capturando un mercado de 8 millones de personas
desatendidas antes de que 3 competidores lancen productos similares en 6-9 meses.

## 2. Contexto y Motivacion

### 2.1 El Problema Hoy

Los instrumentos de renta fija en Colombia (CDTs, bonos) requieren montos minimos
de $1M-$10M COP y procesos presenciales en oficinas bancarias. Esto excluye al 78%
de los colombianos con capacidad de ahorro (fuente: Banco de la Republica, encuesta
financiera 2025).

No existe una plataforma digital que permita micro-inversion en renta fija con
montos accesibles y proceso 100% digital.

### 2.2 Costo de No Resolver

- **Revenue no capturado:** $2.4B COP/ano en comisiones
  (8M personas x ticket promedio $300K x comision 0.1%)
- **Competencia:** 3 fintechs en desarrollo activo, ventana de 6-9 meses
- **Regulatorio:** Sandbox SFC aprobado Q1 2026, pierde validez si no se usa
- **Marca:** Perder posicion de innovador fintech en el mercado colombiano

### 2.3 Por Que Ahora

- Sandbox regulatorio SFC aprobado — ventana de operacion con requisitos reducidos
- Competidores aun no han lanzado — first-mover advantage
- Equipo disponible con experiencia en .NET/AWS
- Presupuesto aprobado ($1.200M COP)

### 2.4 Metricas de Exito

| Metrica | Valor actual | Objetivo | Plazo |
|---|---|---|---|
| Usuarios registrados | 0 | 10.000 | 3 meses post-lanzamiento |
| Inversiones completadas/dia | 0 | 100 | 6 meses post-lanzamiento |
| Ticket promedio | N/A | $300.000 COP | 6 meses post-lanzamiento |
| Tiempo de onboarding (KYC) | N/A | < 5 minutos | Lanzamiento |
| Disponibilidad | N/A | 99.5% mensual | Lanzamiento |

## 3. Opciones Consideradas

### Opcion A: No Hacer Nada

- **Descripcion:** Mantener el estado actual, no entrar al mercado de micro-inversion.
- **Pros:** Sin costo, sin riesgo de cambio, equipo disponible para otros proyectos.
- **Contras:** Pierde ventana competitiva, sandbox SFC sin uso, $2.4B COP/ano no capturado.
- **Estimacion:** $0 COP
- **Por que se descarta:** El costo de oportunidad ($2.4B COP/ano) y la ventana competitiva
  hacen que no actuar sea la opcion mas riesgosa.

### Opcion B: White-Label de Tercero

- **Descripcion:** Licenciar una plataforma de inversion existente (ej: Floid, Simetrik)
  y personalizarla con marca propia.
- **Pros:** Time-to-market rapido (2-3 meses), menor riesgo tecnico inicial.
- **Contras:**
  - Vendor lock-in critico — dependencia total del proveedor para compliance SFC
  - Costo recurrente alto ($50-100M COP/ano en licencias)
  - Customizacion limitada — integracion con Deceval/PSE depende del vendor
  - No hay proveedor certificado para sandbox SFC 2026 especificamente
- **Estimacion:** $400M COP (primer ano: licencia + integracion)
- **Por que se descarta:** Ningun vendor tiene certificacion para sandbox SFC 2026.
  La dependencia regulatoria de un tercero es un riesgo inaceptable.

### Opcion C: Plataforma Propia — RECOMENDADA

- **Descripcion:** Construir plataforma propia con stack .NET 8 + AWS, integraciones
  directas con Deceval (custodia), PSE (pagos) y core bancario (KYC).
- **Pros:**
  - Control total sobre compliance y regulacion SFC
  - Sin vendor lock-in — integraciones directas con proveedores oficiales
  - Costo predecible (equipo interno + infra AWS)
  - Flexibilidad para iterar rapido en fase de sandbox
- **Contras:**
  - Mayor complejidad tecnica inicial
  - Requiere los 6 meses completos hasta Q4 2026
  - Equipo de 5 personas puede ser justo para el scope
- **Estimacion:** $1.200M COP (6 meses de desarrollo)
- **Por que se recomienda:** Es la unica opcion que garantiza control total sobre compliance SFC,
  cabe en el presupuesto aprobado, y el equipo tiene la experiencia necesaria (.NET + AWS).

## 5. Impacto y Riesgos

### 5.1 Sistemas y Modulos Afectados

- **Core bancario:** Integracion nueva para KYC — requiere API de consulta de clientes
- **Deceval:** Integracion nueva para custodia de CDTs — API REST con latencia variable
- **PSE/ACH:** Integracion para pagos — protocolo estandar ya conocido por el equipo
- **SFC:** Nuevo flujo de reportes diarios — batch nocturno

### 5.2 Riesgos con Plan de Mitigacion

| Riesgo | Probabilidad | Impacto | Mitigacion |
|---|---|---|---|
| SLA de Deceval insuficiente para UX deseada | Media | Alto | Patron event-driven con SQS para desacoplar la latencia (1-30s) del flujo del usuario |
| Equipo de 5 insuficiente para scope completo | Media | Alto | MVP estricto: solo CDTs, solo web, solo COP. Scope OUT explicito en CB-001 |
| Cambio regulatorio SFC durante desarrollo | Baja | Alto | Arquitectura modular: compliance como bounded context aislado |
| Integracion KYC con core bancario demora mas de lo esperado | Media | Medio | Spike de 2 semanas con API del core bancario antes de Sprint 1 |
| Datos personales expuestos por vulnerabilidad | Baja | Critico | STRIDE threat model obligatorio, SAST en CI, zero vulnerabilidades criticas como FF |

### 5.3 Plan de Rollback

Si la plataforma no cumple requisitos SFC antes de Q4 2026:
1. Desactivar acceso publico (feature flag de kill-switch)
2. Notificar usuarios registrados en sandbox
3. Migrar datos financieros a core bancario (procedimiento definido en RO-001)
4. Reportar cierre de sandbox a SFC dentro de 5 dias habiles

Rollback es ejecutable porque el sandbox limita a un maximo de 1.000 usuarios
y $500M COP en inversiones activas.

### 5.4 Impacto Regulatorio

- Ley 964/2005: Requiere registro ante SFC — ya completado via sandbox
- Ley 1581/2012: Datos personales — KYC, portafolio, transacciones son datos sensibles
- Circular Basica SFC: Reporte diario obligatorio — batch nocturno automatizado
- Retencion: 10 anos minimo para operaciones financieras

## 6. Preguntas Abiertas

- [x] SLA de Deceval para API REST — confirmado: 99.5% disponibilidad, latencia 1-30s
- [x] Limite maximo de inversion por usuario en fase sandbox — $10.000.000 COP acumulado (validado con Compliance 2026-04-29)
- [x] Frecuencia de reportes SFC — confirmado: diario, batch nocturno

## 7. Supuestos Explicitos

- SUPUESTO-01: El sandbox SFC permite hasta 1.000 usuarios y $500M COP en inversiones activas
- SUPUESTO-02: El core bancario tiene API REST disponible para consulta KYC
- SUPUESTO-03: El equipo de 5 personas esta 100% dedicado al proyecto por 6 meses
- SUPUESTO-04: AWS es el cloud provider (restriccion de equipo, no requiere ADR separado)
