# Context Map: Plataforma de Inversion en Renta Fija

> Referencia: Evans, E. *Domain-Driven Design.* 2003. Capitulo 14: Context Map.
> Referencia: Vernon, V. *Domain-Driven Design Distilled.* 2016. Capitulo 4.

## Bounded Contexts Identificados

| Bounded Context | Subdominio | Tipo | Responsabilidad |
|---|---|---|---|
| **Portafolio** | Core | Stream-aligned | Gestion de inversiones, CDTs, rendimientos, portafolio del usuario |
| **KYC/Identidad** | Supporting | Stream-aligned | Onboarding, verificacion de identidad, SARLAFT, gestion de usuarios |
| **Custodia** | Supporting | ACL hacia Deceval | Registro y consulta de custodia de valores en Deceval |
| **Pagos** | Supporting | ACL hacia PSE | Transferencias de fondos via PSE/ACH, reembolsos |
| **Compliance** | Generic | Complicated-subsystem | Audit trail inmutable, reportes SFC, retencion de datos |

## Relaciones entre Contexts

| Upstream | Downstream | Patron | Descripcion |
|---|---|---|---|
| **KYC/Identidad** | **Portafolio** | Customer-Supplier | KYC provee estado de verificacion; Portafolio consume para autorizar operaciones |
| **Portafolio** | **Custodia** | Customer-Supplier | Portafolio solicita registro de CDT; Custodia ejecuta en Deceval |
| **Portafolio** | **Pagos** | Customer-Supplier | Portafolio solicita transferencia; Pagos ejecuta en PSE |
| **Portafolio** | **Compliance** | Published Language | Portafolio emite eventos de dominio; Compliance los consume para audit trail |
| **KYC/Identidad** | **Compliance** | Published Language | KYC emite eventos de verificacion; Compliance los registra |
| **Custodia** | **Compliance** | Published Language | Custodia emite eventos de registro; Compliance los registra |
| **Pagos** | **Compliance** | Published Language | Pagos emite eventos de transferencia; Compliance los registra |
| **Custodia** | **Deceval (externo)** | Anti-Corruption Layer | ACL traduce modelo interno a API Deceval v3.2 |
| **Pagos** | **PSE (externo)** | Anti-Corruption Layer | ACL traduce modelo interno a protocolo PSE estandar |
| **KYC/Identidad** | **Core Bancario (externo)** | Conformist | KYC se adapta al modelo del Core Bancario para consulta de clientes |

## Domain Events entre Contexts

| Evento | Productor | Consumidor(es) | Payload principal | Transporte |
|---|---|---|---|---|
| `UsuarioRegistrado` | KYC/Identidad | Compliance | userId, timestamp, resultado KYC | SQS FIFO |
| `KYCAprobado` | KYC/Identidad | Portafolio | userId, nivel de verificacion | SQS FIFO |
| `KYCRechazado` | KYC/Identidad | Compliance | userId, motivo de rechazo | SQS FIFO |
| `InversionSolicitada` | Portafolio | Custodia, Pagos, Compliance | inversionId, userId, monto, CDT, idempotencyKey | SQS FIFO |
| `PagoConfirmado` | Pagos | Portafolio, Compliance | inversionId, transaccionPSE, monto | SQS FIFO |
| `PagoRechazado` | Pagos | Portafolio, Compliance | inversionId, motivo | SQS FIFO |
| `CustodiaRegistrada` | Custodia | Portafolio, Compliance | inversionId, codigoDeceval | SQS FIFO |
| `CDTVencido` | Portafolio | Custodia, Pagos, Compliance | inversionId, montoCapital, montoRendimiento | SQS FIFO |
| `RedencionCompletada` | Custodia | Portafolio, Pagos, Compliance | inversionId, montoTotal | SQS FIFO |
| `ReporteSFCGenerado` | Compliance | (ninguno) | reporteId, fecha, totalOperaciones | Log |

## Reglas de Contexto

1. **Ningun bounded context accede directamente a la base de datos de otro.** La comunicacion es siempre via eventos de dominio (SQS FIFO) o API interna.

2. **Los sistemas externos siempre tienen ACL.** Custodia y Pagos encapsulan las APIs de Deceval y PSE — el resto de la plataforma no conoce los detalles de esas integraciones.

3. **Compliance es consumidor pasivo.** Recibe eventos de todos los contextos pero nunca produce comandos. Es append-only.

4. **El idioma ubiquo de cada contexto es distinto:**
   - Portafolio habla de "inversiones", "CDTs", "rendimientos"
   - Custodia habla de "registros de custodia", "codigos Deceval"
   - Pagos habla de "transferencias", "transacciones PSE"
   - Compliance habla de "eventos de auditoria", "reportes regulatorios"

## Nota para Ingenieria

Este Context Map define los limites a nivel de SA. El equipo de ingenieria decide
como implementar fisicamente estos contextos:
- **Opcion recomendada (MVP):** Modulos dentro de un monolito modular (.NET)
  con separacion por namespace y published language via SQS
- **Opcion futura:** Microservicios si la escala o la organizacion lo justifica (> 200K usuarios, > 2 equipos)
