# Tech Spec: Plataforma de Inversion en Renta Fija

## 1. Stack Tecnologico

| Capa | Tecnologia | Version | Justificacion |
|---|---|---|---|
| Backend API | .NET 8 (ASP.NET Core) | 8.x LTS | Experiencia del equipo (CB-001), LTS hasta nov 2026 |
| Worker | .NET 8 BackgroundService | 8.x | Mismo stack, sin overhead de nuevo lenguaje |
| Base de datos | PostgreSQL (RDS) | 16.x | Open source, JSON support, row-level security |
| Cola de mensajes | Amazon SQS FIFO | -- | ADR-001: event-driven con exactly-once delivery |
| Cache | Amazon ElastiCache (Redis) | 7.x | Sesiones, rate limiting, cache de portafolio |
| Almacenamiento | Amazon S3 | -- | Reportes SFC, documentos KYC, audit trail archivado |
| Compute | AWS ECS Fargate | -- | Sin gestion de servidores, auto-scaling, equipo lo conoce |
| API Gateway | AWS ALB + API Gateway | -- | Rate limiting, WAF, SSL termination |
| Secretos | AWS Secrets Manager | -- | Rotacion automatica de credenciales |
| Monitoring | CloudWatch + X-Ray | -- | Nativo AWS, sin costo adicional de tooling |

## 3. Contratos de Integracion

### Contrato: API de Inversiones -> Cola SQS FIFO

| Campo | Valor |
|---|---|
| Protocolo | Async/Queue (SQS FIFO) |
| Topic/Queue | `inversiones-fifo.fifo` |
| Formato | JSON |
| Idempotencia | MessageDeduplicationId = operacion UUID |
| MessageGroupId | userId (orden garantizado por usuario) |
| Mensaje tipo | `InversionSolicitada`, `RedencionSolicitada`, `KYCSolicitado` |
| DLQ | `inversiones-dlq-fifo.fifo` (max 3 reintentos) |

### Contrato: API de Inversiones -> Clientes (Frontend)

| Campo | Valor |
|---|---|
| Protocolo | REST/HTTPS |
| Base URL | `https://api.plataforma.com/v1` |
| Autenticacion | JWT Bearer (RS256), expiracion 15 minutos |
| Rate limiting | 100 req/min por usuario autenticado |
| Formato | JSON, errores en RFC 7807 Problem Details |
| Versionamiento | URL-based: `/v1/` |

**Endpoints principales:**

| Metodo | Ruta | Descripcion | Auth |
|---|---|---|---|
| POST | `/auth/register` | Registro + inicio KYC | No |
| POST | `/auth/login` | Login, retorna JWT | No |
| GET | `/portafolio` | Portafolio del usuario | JWT |
| POST | `/inversiones` | Solicitar compra de CDT | JWT |
| GET | `/inversiones/{id}/estado` | Estado de una operacion | JWT |
| GET | `/cdts/disponibles` | CDTs disponibles para compra | JWT |

### Contrato: Worker -> Deceval (Sistema Externo)

| Campo | Valor |
|---|---|
| Proveedor | Deceval S.A. |
| Protocolo | REST/HTTPS |
| Autenticacion | API Key + certificado mTLS |
| SLA contratado | 99.5% disponibilidad, latencia 1-30 segundos |
| Timeout configurado | 30 segundos |
| Retry | 3 reintentos con backoff exponencial (2s, 4s, 8s) |
| Fallback | Mensaje a DLQ, alerta P2 a Operaciones |
| Idempotencia | `X-Idempotency-Key` header con UUID de operacion |

### Contrato: Worker -> PSE/ACH (Sistema Externo)

| Campo | Valor |
|---|---|
| Proveedor | ACH Colombia |
| Protocolo | REST/HTTPS (protocolo PSE estandar) |
| Autenticacion | API Key + IP whitelist |
| SLA contratado | 99.9% disponibilidad, latencia 2-15 segundos |
| Timeout configurado | 20 segundos |
| Retry | 3 reintentos con backoff (1s, 2s, 4s) |
| Fallback | Cancelar operacion, notificar usuario |

### Contrato: Worker -> Core Bancario (Sistema Interno)

| Campo | Valor |
|---|---|
| Proveedor | Core bancario interno |
| Protocolo | REST/HTTPS (API interna) |
| Autenticacion | JWT service-to-service |
| SLA esperado | 99.9%, latencia 500ms-3s |
| Timeout configurado | 5 segundos |
| Operaciones | `GET /clientes/{cedula}/kyc`, `GET /clientes/{cedula}/sarlaft` |

### Contrato: Worker -> SFC (Sistema Externo)

| Campo | Valor |
|---|---|
| Proveedor | Superintendencia Financiera de Colombia |
| Protocolo | Batch (SFTP/CSV) |
| Frecuencia | Diario, 02:00 AM COT |
| Formato | CSV con schema definido por SFC |
| Retencion | 10 anos en S3 (CB-001 compliance) |

## 4. Quality Attribute Trade-offs

| Trade-off | Decision | Justificacion |
|---|---|---|
| Consistencia vs Latencia | Eventual consistency para flujo de compra | ADR-001: latencia de Deceval (1-30s) hace inviable sincrono. El portafolio se actualiza cuando Deceval confirma |
| Seguridad vs Complejidad | Alta seguridad (JWT RS256 + mTLS + WAF) | Sistema financiero regulado, el costo de un breach es catastrofico. La complejidad adicional es justificable |
| Simplicidad vs Resiliencia | Worker con DLQ y reintentos | La operacion financiera no puede perderse. Prefiero componente extra (Worker) a perder una inversion |

## 6. Politicas a Nivel de Solucion (SA define politica, SWA implementa)

| Politica | Decision SA | SWA implementa |
|---|---|---|
| Errores API | "Todos los errores siguen RFC 7807 Problem Details" | Middleware de error handling, formato JSON |
| Versionamiento API | "URL-based: /v1/" | Routing, backward compat en /v2/ futuro |
| Autenticacion | "JWT RS256, expiracion 15 min, refresh token 7 dias" | Middleware JWT, rotacion de keys, token refresh endpoint |
| Secretos | "Todos en Secrets Manager, rotacion cada 90 dias" | Integracion con SDK, no hardcodear credenciales |
| Logging | "Centralizado en CloudWatch, correlation-id obligatorio en toda request" | Structured JSON logging, middleware de correlation-id |
| Idempotencia | "Todas las escrituras aceptan Idempotency-Key header" | Storage de keys en Redis (TTL 24h), deduplicacion antes de procesar |
| Audit Trail | "Toda operacion financiera se registra ANTES de ejecutarse (fail-closed)" | Middleware de audit, tabla inmutable en PostgreSQL |
