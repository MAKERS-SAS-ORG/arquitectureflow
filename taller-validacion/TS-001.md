# Tech Spec: Plataforma de Micro-Inversion en CDTs

> **Origen:** CB-001 (restricciones: equipo .NET/AWS, presupuesto $1.200M COP),
> RFC-001 (opcion aprobada), ADR-001 (event-driven con SQS FIFO),
> ADR-002 (PostgreSQL como base de datos principal), PRD-001 (casos de uso y NFRs)
>
> **Estado:** Approved. Todos los TODOs resueltos: PostgreSQL justificado en ADR-002,
> SLA Deceval documentado (99%, latencia < 30s), rotacion JWT via AWS Secrets Manager cada 90 dias.

## 1. Stack Tecnologico

| Capa | Tecnologia | Version | Justificacion (-> ADR) |
|---|---|---|---|
| API Backend | .NET | 8.0 LTS | Skills del equipo (CB-001), ecosistema financiero maduro, soporte LTS hasta Nov 2026 |
| Base de datos | PostgreSQL (RDS) | 16.x | Transaccional fuerte, soporte JSON nativo, extensiones pgcrypto para audit trail. ADR-002 |
| Cola de mensajes | Amazon SQS FIFO | -- | Orden garantizado para operaciones financieras, exactly-once processing, managed. ADR-001 |
| Cache | Redis (ElastiCache) | 7.x | Cache de catalogo de CDTs, sesiones JWT, rate limiting por IP/usuario |
| Almacenamiento | Amazon S3 | -- | Documentos KYC (cedulas, selfies), reportes regulatorios SFC, retencion configurable |
| Compute | AWS ECS Fargate | -- | Serverless containers, escala a cero en baja demanda, dentro de presupuesto CB-001 |
| Load Balancer | AWS ALB | -- | TLS termination, path-based routing, health checks, integracion con WAF |
| Secretos | AWS Secrets Manager | -- | Rotacion automatica cada 90 dias, integracion nativa con ECS y RDS |

### Contenedores

| Investment API | .NET 8 (ECS Fargate) | Endpoints REST para onboarding KYC, compra de CDTs, consulta de portafolio. Validacion de reglas de negocio, autorizacion JWT | API |

| Cola SQS FIFO | Amazon SQS FIFO | Mensajes de operaciones pendientes (compras, redenciones, notificaciones). MessageGroupId por user_id, DLQ configurada | Queue |

| S3 | Amazon S3 | Almacenamiento de documentos KYC (cifrado SSE-S3), reportes regulatorios SFC, backup de audit trail | Storage |

## 3. Contratos de Integracion

> Cada contrato define QUE se intercambia y con que SLA.
> La implementacion (librerias, middleware, serializers) es responsabilidad del SWA.

### Contrato interno: Investment API -> Cola SQS FIFO

| Campo | Valor |
|---|---|
| Protocolo | AWS SDK (async) |
| Tipo | Mensajeria asincrona FIFO |
| Formato | JSON |
| MessageGroupId | user_id (garantiza orden por usuario) |
| Idempotencia | Idempotency-Key (UUID v4) como MessageDeduplicationId |
| DLQ | Configurada con max 3 reintentos, backoff exponencial (1s, 4s, 16s) |
| Tipos de mensaje | InversionSolicitada, RedencionProgramada, NotificacionRequerida |
| Monitoreo | Alarma CloudWatch si queue depth > 200 mensajes |

### Contrato interno: Clientes -> Investment API (REST)

| Campo | Valor |
|---|---|
| Protocolo | REST/HTTPS (TLS 1.3) |
| Autenticacion | JWT RS256, expiracion 15 minutos, refresh token 7 dias |
| Endpoints principales | POST /v1/usuarios (onboarding), POST /v1/inversiones (compra), GET /v1/portafolio, GET /v1/inversiones/{id} |
| Respuesta a creacion | 202 Accepted {id, estado: "Recibida", correlation_id} (async por ADR-001) |
| Rate limiting | 100 req/min por usuario autenticado, 20 req/min por IP anonima |
| NFR aplicable | QA-001 del PRD-001: latencia < 500ms p99 |
| Formato de errores | RFC 7807 Problem Details |
| Headers requeridos | Authorization, Idempotency-Key (en POST/PUT), X-Correlation-Id |

### Contrato externo: Worker -> Deceval (Custodia)

| Campo | Valor |
|---|---|
| Proveedor | Deceval S.A. |
| Protocolo | REST/HTTPS con mTLS (certificado cliente) |
| SLA contratado | 99% disponibilidad en horario habil (L-V 7:00-18:00 COT) |
| Timeout configurado | 30 segundos |
| Latencia esperada | < 10 segundos para operaciones de registro de titulo |
| Patron de integracion | Anti-Corruption Layer: modelo interno de Inversion mapeado a esquema Deceval |
| Fallback | Enqueue para reintento en T+5min (max 3 reintentos), usuario ve estado "En Proceso" |
| Idempotencia | Numero de operacion interno como clave de idempotencia |
| Monitoreo | Circuit breaker: abierto despues de 5 fallos consecutivos, half-open cada 60s |

### Contrato externo: Worker -> PSE/ACH (Pagos)

| Campo | Valor |
|---|---|
| Proveedor | ACH Colombia (PSE para debitos, ACH para creditos) |
| Protocolo | REST/HTTPS |
| SLA contratado | 99.5% disponibilidad |
| Timeout configurado | 20 segundos |
| Patron de integracion | Conformist: nos adaptamos al protocolo y esquema de PSE |
| Fallback | Reintento automatico x2 con backoff (5s, 15s), luego notificar al usuario |
| Idempotencia | transaction_id unico por operacion |
| Monitoreo | Alerta si tasa de error > 5% en ventana de 5 minutos |

### Contrato externo: Investment API -> Core Bancario

| Campo | Valor |
|---|---|
| Proveedor | Core Bancario interno |
| Protocolo | REST/HTTPS |
| SLA contratado | 99.9% disponibilidad |
| Timeout configurado | 5 segundos |
| Uso | Consulta de saldos, conciliacion diaria de operaciones |
| Patron de integracion | Anti-Corruption Layer: modelo interno independiente del esquema del core |
| Fallback | Cache de ultimo saldo conocido (TTL 5min), marcar como "saldo aproximado" |

### Contrato externo: Worker -> SFC (Reporte regulatorio)

| Campo | Valor |
|---|---|
| Proveedor | Superintendencia Financiera de Colombia |
| Protocolo | Batch SFTP |
| Frecuencia | Diario a las 23:00 COT (reporte de operaciones del dia) |
| Formato | Archivo plano segun formato definido por Circular Basica SFC |
| Autenticacion | Certificado digital + usuario SFTP |
| Fallback | Reintento en T+1 hora, alerta a Operaciones si falla 2 veces consecutivas |
| Retencion | Archivos generados almacenados en S3 con retencion de 10 anos |

## 4. Quality Attribute Trade-offs

> Trade-offs documentados como decisiones del SA. Cada uno referencia
> el NFR del PRD o la restriccion del CB que lo origina.

| Trade-off | Decision | Justificacion | Origen |
|---|---|---|---|
| Consistencia vs Latencia | Eventual consistency con feedback inmediato | El usuario recibe 202 "Recibida" de inmediato. El Worker procesa async (PSE, Deceval). El usuario ve transiciones de estado: Recibida -> En Proceso -> Confirmada. Mejor UX que bloquear 30s+ esperando respuesta de sistemas externos | ADR-001, QA-001 y QA-002 (PRD-001) |
| Seguridad vs Complejidad | Alta seguridad, se acepta mayor complejidad operativa | mTLS con Deceval, JWT RS256 con rotacion, audit trail fail-closed, cifrado en reposo y transito. La regulacion financiera no permite atajos en seguridad | QA-003 (PRD-001), CB-001 (compliance), RN-004 (PRD-001) |
| Costo vs Escalabilidad | ECS Fargate (pay-per-use) con auto-scaling | Escala a cero en horario no habil, auto-scaling basado en queue depth para Worker y request count para API. Dentro de presupuesto CB-001 ($1.200M COP) | CB-001 (restriccion presupuestal) |
| Simplicidad vs Resiliencia | Event-driven con SQS FIFO + DLQ | Resiliente a fallos de proveedores externos (Deceval, PSE). Las operaciones no se pierden. Agrega complejidad en orquestacion pero elimina riesgo de perdida de transacciones financieras | ADR-001, CB-001 (sistemas externos) |

| FF | Atributo | Que valida | Umbral | Automatizacion | Origen |

| FF-002 | Reliability | Profundidad de cola SQS FIFO | < 200 mensajes en ventana de 15min | Alarma CloudWatch, metrica publicada cada 1min | QA-002 (PRD-001) |

## 6. Politicas a Nivel de Solucion (SA define politica, SWA implementa)

> El SA decide la POLITICA. El Software Architect y el equipo de ingenieria
> deciden la IMPLEMENTACION (frameworks, librerias, configuracion).

| Politica | Decision SA | SWA implementa |
|---|---|---|
| Errores API | "Todos los errores siguen RFC 7807 Problem Details. Incluir type, title, status, detail, instance. Nunca exponer stack traces ni detalles internos en produccion" | Framework middleware, mapeador de excepciones, formato exacto de respuesta |
| Autenticacion | "JWT RS256 con expiracion 15 minutos, refresh token 7 dias. MFA obligatorio en operaciones financieras (compra, redencion). Signing keys rotadas cada 90 dias via AWS Secrets Manager" | Auth middleware, token refresh flow, integracion con Secrets Manager |
| Secretos | "Todos los secretos en AWS Secrets Manager. Rotacion automatica cada 90 dias. Zero secretos en codigo, variables de entorno, o archivos de configuracion" | Integracion SDK, config local para desarrollo, politicas IAM |
| Idempotencia | "Todas las escrituras (POST, PUT) aceptan header Idempotency-Key (UUID v4). Ventana de deduplicacion: 24 horas. Respuesta identica para requests duplicados" | Storage de keys en Redis (TTL 24h), middleware de deduplicacion |
| Correlation ID | "Todas las requests llevan header X-Correlation-Id. Si el cliente no lo envia, el API lo genera (UUID v4). Se propaga a SQS, Workers, logs, y audit trail" | Middleware de propagacion, structured logging |
| Audit trail | "Inmutable, append-only, hash encadenado (SHA-256). Retencion 10 anos. Principio fail-closed: si el audit trail falla, la operacion financiera NO se ejecuta" | Implementacion de hashing, tabla en PostgreSQL, backup a S3 |
| Versionamiento API | "URL-based: /v1/. Backward compatibility obligatoria por minimo 6 meses despues de deprecacion. Comunicar deprecaciones via header Sunset" | Routing config, versionado de controllers, header Sunset |
| Rate limiting | "100 req/min por usuario autenticado, 20 req/min por IP anonima. Respuesta 429 con header Retry-After" | Sliding window en Redis, middleware en ALB/API |
| Cifrado | "En transito: TLS 1.3 obligatorio. En reposo: AES-256 para RDS y S3. Documentos KYC cifrados con SSE-S3" | Configuracion de certificados, politicas de cifrado RDS/S3 |

## 7. Decisiones Resueltas

> Todos los TODOs originales han sido resueltos en esta version Approved:

| Decision | Resolucion | Referencia |
|---|---|---|
| Base de datos PostgreSQL vs alternativas | PostgreSQL seleccionado por soporte transaccional fuerte, JSON nativo, extensiones pgcrypto para audit trail, costo operativo bajo en RDS | ADR-002 |
| SLA Deceval sin documentar | SLA confirmado: 99% disponibilidad en horario habil, latencia < 10s para registro de titulo. Timeout configurado a 30s con margen. Circuit breaker como proteccion adicional | Contrato Deceval firmado |
| Rotacion de JWT signing keys | Rotacion automatica cada 90 dias via AWS Secrets Manager. Key pairs RS256. El API soporta multiple keys activas durante periodo de rotacion (grace period 24h) | Politica de seguridad aprobada |
